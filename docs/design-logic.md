# 设计逻辑

## 原型定位

本网页不是手机地图，也不是商业旅游 App。它模拟的是杭州桥西历史文化街区中可被更新的实体电子导览牌界面，即 Inclusive Wayfinding Totem。

## 系统关系

```text
现有电子导览牌
↓
交互式 Totem
↓
历史节点等级
↓
面向不同身体与认知需求的路线
↓
地图、语音、双语与无障碍辅助
```

## 信息层级

一级节点：
街区结构与历史叙事核心，例如桥、运河岸线、工业遗产展馆入口。

二级节点：
街巷转折、公共空间、视线转换点，用于帮助用户理解空间序列。

辅助节点：
休息点、方向确认点、无障碍提示点，用于支持低视力用户、视觉障碍用户、老年用户与初访者。

## 用户模式

普通游客：
强调完整的历史文化叙事与空间体验。

低视力用户：
强调高对比、少转折、大字体、清晰边界。

视觉障碍用户：
强调语音优先、节点确认、方向转折提示。

老年用户：
强调短距离、低强度、更多休息点和简明信息。

初次到访游客：
强调先建立整体方位感，再进入重点历史节点。

## 后续扩展建议

- 将示意点位替换为真实测绘坐标。
- 增加 Totem 安装位置与朝向字段。
- 增加路径坡度、铺装、台阶、过街风险等无障碍属性。
- 增加触摸屏大按钮模式和屏幕下半区优先交互布局。
- 增加建筑设计汇报中的系统剖面图、节点轴测图或服务蓝图。
